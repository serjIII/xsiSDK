Localization

You can localize the plug-in after it is internationalized.

The following steps are involved in localization:

  1. Create a dictionary (an XML file) with the .dictxml file extension. See Creating an XML File.
  2. Add the strings that need to be localized to this file. See Adding Strings to the File.
  3. Compile the .dictxml file into a binary file (.dict) using xsiDictParser.pl. See Compiling to a Binary File.
  4. Deploy the dictionary. See Deploying the Dictionary.

You can also refer to the InternationalPSetUI plug-in example located in the SDK workgroup (<yourinstallationfolder>XSISDK\examples\workgroup\Addons) for more information.

Creating an XML File

You must create the XML file in the unicode format.

NOTE:It is a best practice to name the XML file as EXT<pluginname>.dictxml, where EXT represents an external file.

Following is an example template for the .dictxml file: 

<?xml version='1.0' encoding='UTF-8'?>
<xsidict>
    <namespace name="EXTFooPlugin" headerfile="sidictres.h" langs="en,jp">
        <dictstring context="…">
            <dictkey>…</dictkey>
            <dictval lang=”jp”>…</dictval>
        </dictstring>
        ... 
        <dictstring context="…" unique_lang="1">
            <dictkey>…</dictkey>
        </dictstring>
        ...
        <dictidstring define="IDS_XXX" idvalue="…" context="…">
            <dictval>…</dictval>
            <dictval lang=”jp”>…</dictval>
        </dictstring>
        ...
        <dictidstring define="IDS_XXX" idvalue="…" context="…" unique_lang="1">
            <dictval>…</dictval>
        </dictstring>
    </namespace>
</xsidict>
The following table defines the elements and attributes in the .dict file.
Element Description
xsidict Root element. There is only root element in each file.
namespace Namespace. Following are the related attributes:

  • name – The unique identifier for a dictionary.
  • headerfile – Optional. The .h header file is generated including all the string declarations referenced by the numeric ID.
  • langs – The languages that are translated.

dictstring The string entry that is referenced by the English version of the string. Following are the related attributes:

  • context – Additional description for this string. This attribute is not compiled. It is used to provide extra information when editing.
  • unique_lang – Add this attribute if the string need not be translated. You do not need to add the Japanese text.

Folowing are the child elements:

  • dictkey – The English string that is used as key in the string table.
  • dictval – The string for a specific language. The lang attribute is used for specifying the language. If this attribute is not specified, the string is considered to be in English.
dictidstring The string entry that is referenced by its numeric ID. Following are the related attributes:

  • define – The ID representation to be referenced.
  • idvalue – Allocate a unique numeric ID.
  • context – Same as the context attribute for the dictstring element.
  • unique_lang – Same as the unique_lang attribute for the dictstring element.

The dictval child element is same as the child element for dictstring.

NOTE: Do not define the dictkey child element for dictidstring.

Adding Strings to the File

You can add new strings in the following formats:

  • dictstring 
    <dictstring context="Write where to find the string in the UI">
    <dictkey>Add the English string here</dictkey>
    <dictval lang="jp">Add the Japanese string here</dictval>
</dictstring>
  • dictidstring 
    <dictidstring define="IDS_MYNEWSTRING" idvalue="1" context="Write where to find the string in the UI">
    <dictval>Add the English String here</dictval>
    <dictval lang="jp">Add the Japanese string here</dictval>
</dictidstring>

Follow these guidelines:

  • You must use a unique idvalue. If you repeat the idvalue, it results in a compiler warning message.
  • You must specify the context. If you do not specify the context, it results in a compiler error. Information in the context is helpful to the translator. You can specify where to find a string in the UI or how to use it in the context.
  • If you do not want to translate a string, remove the Japanese text and use the unique_lang=”1” flag as shown in the following example. 
    <dictidstring define="IDS_MYNEWSTRING" idvalue="1" unique_lang=”1” context="Write where to find the string in the UI">

Compiling to a Binary File

  1. Open the Softimage command prompt.
  2. Run the following command: 
    xsiDictParser.pl –dictdir C:\Program Files\Autodesk
\Softimage <version>\Application\Dictionary –inputfile “EXTFooPlugin.dictxml”

You can specify the –dictdir directory as <your_installation_folder>\Application\Dictionary.

The EXTFooPlugin.dict file is generated in the en and jp subfolders under the –dictdir directory.

Deploying the Dictionary

Store each .dict file in a language folder (en and jp) at <your_installation_folder>\Application\Dictionary\<language_folder>. For example, the .dict file for English must be saved in the en folder, and the .dict file for Japanese must be saved in the jp folder.

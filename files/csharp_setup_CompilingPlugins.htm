Compiling Plug-ins

TIP:If you are new to .NET, you can refer to the Working with the Global Assembly Cache (GAC) section for some useful information.

To compile a plug-in:

  1. Open a Softimage command prompt, and type devenv to start Visual Studio.

    Starting Visual Studio from a Softimage command prompt sets the environment variables such as XSI_HOME properly, which avoids the build and link errors.

  2. In Visual Studio , open the .csproj project file.

    If you want to load the plug-in from a network share, make sure to create a strong name key for your plug-in.

  3. Select a configuration (Debug or Release) and build the DLL.

    The first time you build the plug-in, you are prompted to save a Solution File (.sln).

  4. To load the plug-in from a network share, make sure to add the plug-in assembly to the GAC (Global Assembly Cache) by running the following from a command prompt: 

    gacutil /i \servername\<plugin_path>\YourPluginName.dll

  5. Do one of the following to load the plug-in into Softimage:

    • In the Plug-in Manager, click Update All.

    • In the Plug-in Manager, click Load, locate the plug-in, and click OK.

    • In the Plug-in Manager, right-click the Plug-ins folder, and select Update from the context menu.

Working with the Global Assembly Cache (GAC)

The Global Assembly Cache (GAC) contains assemblies that can run across multiple applications on a single machine. It performs integrity checks when assemblies are added to the cache, and uses strong names to load the correct version.

You can manage the cache with the Global Assembly Cache Utility (gacutil). You can install and uninstall assemblies in the cache as well as list the installed assemblies using the gacutil. To use the gacutil, you must install the .NET Framework SDK, which is available from the MSDN Framework Developer Center.

For Softimage, you must determine whether the Softimage SDK assemblies are installed and which versions are running. If they are not installed, you need to install them.

To determine whether the Softimage assemblies are installed:

  • Run these commands from the Softimage command prompt: 

    gacutil /l Softimage.XSIOM
gacutil /l Softimage.XSIUtil
gacutil /l Softimage.XSIMath
    NOTE:You might see the following message when you run this utility: 
    'gacutil' is not recognized as an internal or external command, operable program or batch file.

    Check the C:\Program Files (x86)\Microsoft SDKs directory and add the folder to your %PATH%.

If they are properly installed, you might see the following: 

The Global Assembly Cache contains the following assemblies:
	Softimage.XSIMath, Version=1.0.0.0, Culture=neutral, PublicKeyToken=a90401fcf6bc5571, processorArchitecture=MSIL
Number of items = 1

The Global Assembly Cache contains the following assemblies:
	Softimage.XSIUtil, Version=1.0.0.0, Culture=neutral, PublicKeyToken=a90401fcf6bc5571, processorArchitecture=MSIL
Number of items = 1

The Global Assembly Cache contains the following assemblies:
	Softimage.XSIMath, Version=1.0.0.0, Culture=neutral, PublicKeyToken=a90401fcf6bc5571, processorArchitecture=MSIL
Number of items = 1

If they are not installed properly, the following message appears: 

The Global Assembly Cache contains the following assemblies:

Number of items = 0

To install the Softimage assemblies in the GAC:

  • Open a Softimage command prompt (it automatically opens at the Application/bin directory under the factory location) and enter the following three commands: 

    gacutil /i Softimage.XSIOM.dll
gacutil /i Softimage.XSIUtil.dll
gacutil /i Softimage.XSIMath.dll

    If the installation is successful, the following message appears: 

    Assembly successfully added to the cache

To uninstall the Softimage assemblies from the GAC

  • Open a Softimage command prompt (it automatically opens at the Application/bin directory under the factory location) and enter the following three commands: 

    gacutil /u Softimage.XSIOM.dll
gacutil /u Softimage.XSIUtil.dll
gacutil /u Softimage.XSIMath.dll

Strong Names

Strong names are encoded signatures that keep assemblies unique on a system. To add your assembly to the GAC, you need to use a strong name key, often stored in a file (.snk).

To create a strong name key:

  1. With your .csproj project file open in Visual Studio, select <YourProjectName> Properties from the Project menu.

  2. Click the Signing tab, and then select the Sign the assembly check box.

  3. From the Choose a strong name key file drop-down list, select <New>. The Create Strong Name Key dialog appears.

  4. Enter a strong name key and click OK.

Using C# with Softimage

C# (pronounced C sharp) is a simple, type-safe programming language that represents a hybrid between C++ and scripting languages. Plug-ins written in C# are much faster than the scripted plug-ins.

The Microsoft Developer's Network (MSDN) has technical specifications, C# reference, and overview documentation about the C# language. In addition, MSDN offers a C# FAQ and Microsoft's GotDotNet provides a number of QuickStart tutorials.

TIP:If you do not have Visual Studio, you can use Visual C# Express, a free version that comes with full C# doc and a debugger.

How C# Works with Softimage

The Assembly is the primary unit of deployment within the .NET Framework. It is similar to a C++ .dll or .so file and can contain references to other assemblies and modules.

The Softimage Object Model type library is ported to a C# assembly, which allows developers to access the Softimage SDK using C#. The entire object model is accessible, except for a few interfaces as follows:

  • Obsolete Interfaces – XSIDialog, Cluster_V1, ClusterProperty_V1, Geometry_V1, Geometry0D, Geometry1D, and Geometry2D

  • Empty Interfaces – AlignedAxisConstraint and SpotLight

  • PPG (handled by the class for C#) and XSIFileService (import/export services)

    NOTE:You can familiarize the Softimage Object Model for C# using Visual Studio features such as the ObjectBrowser and autocomplete function. You can use ObjectBrowser to browse through the Softimage object model typelib. The autocomplete function provides a list of available objects and their members as you type your source code.

How C# Plug-ins Work with Softimage

Instead of functions exported from a .dll, C# plug-in items are implemented as C# objects. You can create a plug-in object that implements public methods matching the plug-in item callbacks. You must make sure to include the XSIOM (object model) assembly and then create a base object that registers the plug-in in Softimage. For more information, see Building Softimage Plug-ins with C#.

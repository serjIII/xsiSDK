Specifying a Help File for a Plug-in

You can specify an html page as the help file for a plug-in.

You can access the help for a plug-in in the following ways:

  • In the Plug-in Tree, right-click the plug-in and choose Help.

  • If the plug-in includes a self-installing property, click the ? icon in the title bar of the property page.

NOTE:Pressing F1 in the script editor does not open the help for a custom command. This action works only for commands, objects, methods, and properties that are described in the Softimage SDK help.

By default, Softimage looks for a help file with the same name as the plug-in in a Doc subfolder. For example, consider that the plug-in MyPlugin.js is located in the following folder:

C:\users\xyz\Autodesk\Softimage_<version>\Application\Plugins\MyPlugin

In this example, Softimage looks for the help file named MyPlugin.html or MyPlugin.htm in the MyPlugin\Doc subfolder.

If a plug-in help file has a different name or location, you can use SIObject.Help, PluginRegistrar::PutHelp(), or PluginItem::PutHelp() to specify the help file. See the following example: 

// JScript
function XSILoadPlugin(in_reg)
{
	in_reg.Author = "xyz";
	in_reg.Name = "MyPlugin";
	
	// Help file is in the same folder as the plug-in
	in_reg.Help = XSIUtils.BuildPath(in_reg.OriginPath, "MyPlugin.htm");
	
	var oPluginItem = in_reg.RegisterProperty("MyProperty");
	
	// Open a specific help topic for the property
	oPluginItem.Help = XSIUtils.BuildPath(in_reg.OriginPath, "MyProperty.htm");
	
	//RegistrationInsertionPoint - do not remove this line
	
	return true;
}
NOTE:For html pages, you cannot include a destination anchor. For example, MyPlugin.htm#command.

Customizing the Plug-In Help Icon

You can customize the help icon that appears in the property page of a plug-in. The help icon must be a bitmap file (*.bmp). It must have a height of 18 pixels and a width not exceeding 100 pixels. If the help icon does not meet this criteria, the default ? icon is displayed in the property page. By default, Softimage looks for the help icon file in a Doc subfolder inside the plug-in installation folder.

You can use one of the following to specify a help icon file for the help button in a property page:

  • PluginItem.HelpButton or PluginItem.PutHelpButton
  • PluginItemRegistrar.HelpButton or PluginItemRegistrar.PutHelpButton

See the following example: 

' VBScript
'
' This example shows how to specify a help icon file for a plug-in 
' and specific plug-in items. The help icon files are assumed
' to be in a "doc" subfolder inside the plug-in installation folder.
'
	
Function XSILoadPlugin( in_reg )
	' Register plugin information
	in_reg.Author = "Autodesk Softimage" 
	in_reg.Name = "Help Button"
	in_reg.Major = 1
	in_reg.Minor = 0 
	in_reg.Help = "Help_Button.htm"
	in_reg.HelpButton = "Help_Button.bmp"
	
	' No need to set the help icon file for this property,
	' the "MyFirstProperty_HelpButton.bmp" file located in the
	' "doc" subfolder is automatically found by Softimage
	set item = in_reg.RegisterProperty( "MyFirstProperty" )
	'item.HelpButton = "MyFirstProperty_HelpButton.bmp"

	' This help icon file is in the same folder as the plug-in
	set item = in_reg.RegisterProperty( "MySecondProperty" )
	sHelpButton = "MyHelpButtonFile.bmp"
	item.HelpButton = XSIUtils.BuildPath( in_reg.OriginPath, sHelpButton )

	' No need to set the help icon file for this plug-in item, the help
        ' button file set for the plug-in is suitable for this plug-in
        ' item (considering there is no "MyThirdProperty_HelpButton.bmp" file
        ' located in the "doc" subfolder of the plug-in installation folder)
	
	in_reg.RegisterProperty "MyThirdProperty"
	
	XSILoadPlugin = true
end Function

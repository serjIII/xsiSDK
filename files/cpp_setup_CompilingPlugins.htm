Compiling Plug-ins

About the Softimage Version

The Softimage version information consists of the runtime version and compilation version. The compilation version is used for the compilation check. You can access the runtime version through the Application::GetVersion function, and the compilation version through the XSI_VERSION preprocessor define.

Recommended Setup

  • For Linux, the recommended compiler is gcc-4.1.2.

  • For Windows the recommended development environment is Visual Studio 2012 Update 4.

NOTE:You can create Visual Studio 2010 and Visual Studio 2012 projects from the plug-in wizard.

Compiling Plug-ins in Linux

  1. In a shell (tcsh) window, type:

    source <Softimage_home_directory>/<.xsi_version>

    This action makes sure that the environment variables such as XSISDK_ROOT are set correctly to avoid the build and link errors.

    TIP:In the Plug-ins Tree, right-click the Plug-ins node and select Command Prompt to open an xterm window.

  2. Change the directory to the location of the plug-in source code.

  3. To remove all intermediate files before building the example, run the following command:

    gmake clean

  4. To compile the example, run the following command:

    gmake

  5. Do one of the following to load the plug-in into Softimage:

    • In the Plug-in Manager, click Update All.

    • In the Plug-in Manager, click Load, locate the plug-in, and click OK.

    • In the Plug-in Manager, right-click the Plug-ins folder, and select Update.

Compiling Plug-ins in Windows

You must start Visual Studio with the correct options that are set up using a batch file (.bat).

  1. Set up a 64-bit environment as follows:

    • Save the following code snippet in a batch file (for example, VC64XSIx64.bat).

      NOTE:The path names used in the following code snippet are examples. You must use the actual Softimage installation directory and Visual Studio installation directory in the path names. 
    @echo off

call "C:\Program Files\Autodesk\Softimage <version>\Application\bin\setenv.bat"
call "C:\Program Files (x86)\Microsoft Visual Studio <version>\VC\vcvarsall.bat" amd64

set PATH
set "XSISDK_ROOT = C:\Program Files\Autodesk\Softimage <version>\XSISDK"

"C:\Program Files (x86)\Microsoft Visual Studio <version>\Common7\IDE\devenv.exe"

echo on

    NOTE:You must use the batch files to set up and start Visual Studio so that the environment variables such as XSISDK_ROOT are set properly to avoid the build and link errors. These batch files simulate opening a Softimage command prompt to start Visual Studio.

  2. Open the project file .vcxproj in Visual Studio.

  3. Select Build Configuration Manager.

  4. In the Active solution configuration drop-down list, select either Debug x64 or Release x64.

  5. In the Active solution platform drop-down list, select New, and then type x64 in the New Solution Platform dialog.

  6. Build the DLL.

    You are prompted to save a Solution File (.sln) when you build the plug-in for the first time.

  7. Load the plug-in into Softimage by doing one of the following:

    • In the Plug-in Manager, click Update All.

    • In the Plug-in Manager, click Load, locate the plug-in .DLL, and click OK.

Environment Variables and Build Flags

If you use the Softimage SDK wizards, you do not need to set up the development environment. However, if you want to set up your development environment, this topic provides a quick reference to the required settings.

Windows

Starting Visual Studio

You must start Visual Studio from a Softimage command prompt to make sure that the environment variables such as XSISDK_ROOT are set correctly to avoid the build and link errors.

Project Configuration Properties

The Softimage SDK wizards generate projects with the following configuration properties.

Property

Value

C/C++ > General > Additional Include Directories

$(XSISDK_ROOT)\include

C/C++ > Preprocessor > Preprocessor Definitions

WIN32;NDEBUG;_WINDOWS

or

WIN32;_DEBUG;_WINDOWS

C/C++ > Code Generation > Runtime Library

Multi-threaded DLL (/MDd)

or

Multi-threaded Debug DLL (/MDd)

Linker > General > Additional Library Directories

$(XSISDK_ROOT)/lib/nt-x86-64

Linker > Input > Additional Dependencies

sicppsdk.lib sicoresdk.lib

To make the location of the include and library directories available in all projects, you can add these to the global VC++ directories.

Linux

The Softimage SDK wizards generate GNU makefiles that you can use to build your plug-ins.

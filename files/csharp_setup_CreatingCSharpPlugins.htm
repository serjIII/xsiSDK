Creating C# Plug-ins

The best way to get started with developing a self-installing plug-in is to use one of the Softimage SDK wizards to generate the initial code. The development environment is set up correctly if you use the wizard.

It is a good practice to use Add-on directories to store both the source code and compiled plug-in. For example, in the SDK example workgroup, each example is stored in a separate Add-on directory.

The C# examples put their source code in a cssrc folder.

To generate the C# code for the plug-in

  1. Select File Plug-in Manager. The Plug-in Manager dialog appears.

    NOTE:In the Tools Development Environment layout, you can click the Plug-ins tab to switch to the Plug-in Manager view.

  2. Select File New, and then choose a type of plug-in.

    The Plug-in Wizard appears.

  3. Specify the plug-in details such as plug-in name, author, and others.

  4. In the Coding Language drop-down list, select C#.

  5. In the Output Directory box, select or type the location to create the plug-in.

  6. Generate the code.

    The wizard generates a C# project file (.csproj) and a C# source file (.cs) that contains the code to register and create C# objects.

    NOTE:The wizard outputs the source and project files to the folder specified in the Output Directory box, but Visual Studio compiles the .dll in the obj/Debug subfolder which Softimage finds automatically.

  7. Edit and compile the code.
